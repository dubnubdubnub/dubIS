## Decisions

1. **escapeHtml quote escaping**: The spec says escapeHtml escapes `"` and `'` (stricter than `ui-helpers.escHtml`). Confirmed this creates safe attribute values. The test verifies the output of `escapeHtml()` directly because jsdom decodes `&quot;` and `&#39;` back to literals when reading `.innerHTML` in text content context — the html() function still uses escapeHtml internally, so the behavior is correct.

2. **Node placeholder strategy**: Used `<!--⟦node:N⟧-->` placeholder comments in the markup string, then walked the resulting `template.content` with a TreeWalker (SHOW_COMMENT filter) and replaced each placeholder with the real Node. This preserves Node identity and is framework-free.

3. **PredicateEditor AND/OR toggle threshold**: The spec says "an AND/OR toggle for the group" with no threshold mentioned. Initial implementation hid it for <2 rules. Changed to show it for ≥1 rules so it's always accessible when a rule exists.

4. **bind() key format**: "type selector" split on first space only, so the selector may contain spaces (e.g., "click div .btn"). This matches the spec note "selector may contain spaces? keep simple: split on first space".

5. **predicate-ui.js does not import delegate.js**: The spec says "uses html + delegate" for the PredicateEditor. During implementation, all event wiring was done directly via `addEventListener` on the constructed `el()` elements (since we own them), which is simpler than delegating from the container. The `bind` import was removed because eslint flagged it as unused; `html` itself is imported and used. For future phases where the editor is mounted in a larger container and needs teardown delegation, `bind` can be imported then.
